### 双场景联动说明

本工作流描述的是 **实时通话干预全流程**（场景二），但其首尾两端均深度依赖 **离线非实时数据分析场景**（场景一）：

| 流程环节 | 关联场景一的内容 | 对应文档章节 |
|---|---|---|
| **流程起点** — 调取离线画像 | 场景一产出的客户性格标签、情绪特质、最优沟通时长 | §4.3 客户标签体系生成模块 + 最优沟通时长测算模块 |
| **流程终点** — 数据回流迭代 | 实时数据回流后，反哺场景一的标签精度、时长测算模型、话术库 | §6 双场景数据联动方案 |

以下为完整流程的四个阶段书面拆解。每一个标注了 📡 的步骤代表**场景一离线数据在此处参与交互**。

## 一、场景启动与数据接入阶段

### 1.1 离线画像前置加载

坐席接听通话前，系统自动向离线数据服务层发起请求，调取该客户的**数字化画像档案**。该档案由历史通话数据经离线场景批量挖掘产出，包含以下核心维度：

- **性格标签**：基于多通历史通话的语速特征、表达习惯、质疑频次、情绪反应模式，经聚类算法生成（如直爽型、谨慎型、敏感型、强势型等）；
- **情绪特质**：统计客户历史通话中各类情绪占比，量化其情绪稳定性、负面情绪触发概率及情绪消散速度；
- **最优沟通时长**：通过回归算法拟合客户正面情绪峰值区间与注意力衰减曲线，测算出的个体化高效沟通时长阈值。

上述画像数据均来源于**场景一（§4 离线非实时数据分析）** 的标准化产出。其中性格标签经 §4.3.1 特征阈值匹配 + 聚类算法双重机制生成，最优沟通时长经 §4.3.2 统计回归算法拟合产出。

画像数据在坐席终端前置展示，帮助坐席在开口前即完成**千人千面的沟通预判**，为后续实时干预提供离线知识底座。

### 1.2 实时语音流接入与流式切片预处理

通话建立后，实时语音流以 PCM 编码持续接入系统。系统随即启动**流式预处理管线**，包含三个子步骤：

1. **流式切片**：将连续语音流以 2–5s 为窗口长度进行滑动切片，确保每一帧数据量满足模型推理的实时性要求；
2. **语音降噪**：采用维纳滤波 + 深度学习降噪模型，滤除环境噪音、设备电流声等非人声干扰；
3. **人声分离**：基于 Speaker Verification 技术，从混话信号中分离坐席与客户两条人声支路，**仅保留客户语音流**送入后续特征提取管线，避免坐席语音对情绪判定产生交叉干扰。

## 二、多维度特征提取与模型推理阶段

### 2.1 三维特征并行提取

系统从三个互补维度并行提取客户语音的深层表征，确保情绪判定的鲁棒性与准确性。

#### 2.1.1 声学韵律特征

提取人声物理维度的底层声学参数，不受话术内容影响，真实反映客户生理层面的情绪唤起状态：

| 特征项 | 描述 |
|---|---|
| 音量均值 / 方差 | 反映客户整体响度水平及波动剧烈程度 |
| 语速 | 每秒音节数，情绪激动时语速显著上升 |
| 基频音调 / 语调斜率 | 音调基频均值及上升/下降斜率，愤怒时基频骤升 |
| 呼吸频率 / 停顿频次与时长 | 情绪紧张时呼吸加速、停顿减少 |
| 语音颤音特征 | 高频震颤能量分布，恐惧或暴怒时显著增强 |

#### 2.1.2 语义文本特征

基于 ASR 实时转写文本，通过 NLP 解析提取语义层面的情绪信号：

- **负面关键词词频**：匹配预设负面情感词典，统计单位时间内负面词汇出现密度；
- **反问/感叹句式占比**：检测句式结构，反问与感叹句式通常伴随不满或激动情绪；
- **诉求表达强度**：通过语义角色标注与依存句法分析，量化客户诉求的紧急程度与强制语气；
- **语义抵触度**：分析客户对坐席回复的否定、反驳、质疑语义强度；
- **消极语句密度**：综合计算负面情感语句在总语句中的占比。

#### 2.1.3 时序情绪特征

捕捉情绪在通话时间轴上的动态演化规律：

- **情绪波动曲线**：记录连续时间窗口内的情绪等级变化轨迹，形成可视化曲线；
- **情绪拐点数量**：自动标记情绪等级出现方向性变化的时序节点（如由 2 级跃升至 3 级）；
- **负面情绪持续时长**：统计客户处于 3 级及以上等级的连续持续时间；
- **情绪回升速率**：负面情绪后回归平静所需的时间斜率，反映客户情绪自我调节能力；
- **情绪突变频率**：单位时间内情绪等级跳变 2 级及以上的次数，高频突变为高危前置信号。

### 2.2 CNN+LSTM 混合模型毫秒级推理

上述三个维度的特征在特征融合层完成拼接对齐后，实时送入 **CNN+LSTM 混合深度学习模型**：

- **CNN（卷积神经网络）**：负责在局部时间窗口内提取声学特征与语义特征的高阶组合模式，捕捉短时突变信号；
- **LSTM（长短期记忆网络）**：接收 CNN 输出的特征序列，建模通话全程的时序依赖关系，学习情绪长期演化规律。

模型推理结果以 **毫秒级时延（≤200ms）** 输出两项核心产出：

1. **当前情绪等级**（1–5 级）：平静愉悦、轻微不满、明显烦躁、生气抵触、暴怒失控；
2. **情绪趋势预判**：基于当前时序上下文，预测后续 3–5 个时间窗口的情绪走向（上升 / 持平 / 下降），为前置干预争取时间窗口。

推理结果同步推送至前端**情绪实时可视化模块**，在坐席终端动态绘制**情绪波动曲线**，并标注当前等级与风险标签。

## 三、分级策略干预与实时控制阶段

系统根据当前判定的情绪等级（1–5 级），联动标准化干预策略引擎，自动匹配差异化应对方案。以下为各等级对应的**技术判定特征**与**系统干预策略**。

### 3.1 1 级 — 平静愉悦

**技术判定特征：**
- 语速、音量保持平稳，无明显波动；
- 无负面语义关键词，语调自然；
- 情绪波动值长期趋近于 0。

**系统干预策略：**
- 系统无感知，坐席界面维持常规状态；
- 提示坐席**正常推进业务**，主动引导客户表达诉求，维持当前沟通节奏；
- 无需触发任何预警或干预动作。

### 3.2 2 级 — 轻微不满

**技术判定特征：**
- 语速微快，出现轻微质疑类语句；
- 音量出现小幅波动，负面关键词词频较低（1–2 次 / 窗口）；
- 情绪等级在 1–2 级之间微量震荡。

**系统干预策略：**
- 触发**一级弹窗预警**（非侵入式，浅色提示条）；
- 提示坐席放缓语速、降低语调，主动解答客户疑问；
- 建议坐席向客户表达**提前共情**，防止情绪进一步升级。

### 3.3 3 级 — 明显烦躁

**技术判定特征：**
- 音量明显升高，频繁打断坐席发言；
- 抱怨语句增多，负面关键词词频上升（3–5 次 / 窗口）；
- 情绪波动持续上升，无回落趋势。

**系统干预策略：**
- 触发**二级高亮预警**，坐席界面状态标签变为橙色高亮；
- 建议**暂停业务宣讲**，停止推销或信息输送类话术；
- 系统动态推送**共情安抚话术模板**至话术推荐面板，坐席可一键复制使用。

### 3.4 4 级 — 生气抵触

**技术判定特征：**
- 音量骤升、语调尖锐刺耳；
- 负面语义高度密集，对抗性语句（反驳、指责）占比超过 50%；
- 情绪等级持续稳定在 4 级附近，无明显回落。

**系统干预策略：**
- 触发**三级风险预警**，界面状态标签变为红色高亮；
- 界面**强制提示**「优先安抚 · 禁止争辩」，锁定所有推销类话术的推送；
- 系统停止推荐任何可能激化矛盾的进攻性话术，仅保留安抚与道歉类策略。

### 3.5 5 级 — 暴怒失控

**技术判定特征：**
- 语音急促混乱、语速极快，高声斥责甚至出现咆哮；
- 情绪突变峰值极高，短时间内从基线跃升至 5 级；
- 时序特征表现为明显的情绪失控信号。

**系统干预策略：**
- 触发**最高级紧急预警**，界面状态标签变为**深红色高频闪烁**，营造高度紧迫的可视化警示氛围；
- 系统在后台**毫秒级同步推送主管监听介入**，主管终端自动弹出来接通话的实时情绪数据面板；
- 坐席控制台提供两个一键操作按钮：
  - **转接资深坐席**：将通话一键转接至预先配置的资深处理队列；
  - **延后回电**：结束当前通话，系统自动登记回执并排期后续回访。

## 四、通话结束与数据闭环阶段

### 4.1 通话后自动处理

通话结束后，系统自动执行以下三项后处理操作：

1. **情绪拐点记录**：将本次通话中所有情绪等级方向性变化的时序节点、触发关键词与坐席应对行为写入结构化日志；
2. **高危通话标记**：若通话中出现 4 级或 5 级情绪，系统自动将该通话归类为**高风险工单**，推送给质检团队优先复核；
3. **通话质量评分**：综合本次通话的情绪稳定性、负面情绪占比、坐席干预时效与效果，计算单次通话质量**综合评分（0–100 分）**，归档至客户历史记录。

### 4.2 数据回流与场景一闭环迭代

实时通话产生的全量过程数据（情绪等级序列、拐点日志、干预记录、评分结果）通过数据回流通道同步至**离线数据库（场景一）**，用于三个方向的迭代优化（对应 §6 双场景数据联动方案）：

1. **迭代客户标签精度**（→ §4.3.1）：将本次通话中新采集的情绪数据与性格表现，增量更新至该客户的离线画像标签，持续修正聚类模型输出的标签置信度；
2. **优化最优沟通时长测算模型**（→ §4.3.2）：将本次通话的有效沟通时长及负面情绪触发临界时间节点作为新样本，加入回归训练集，迭代时长测算模型的拟合精度；
3. **丰富风险话术库**：分析本次通话中成功/失败的干预案例，提取有效话术模式与无效策略，更新策略引擎的权重配置与话术模板库。

至此，系统完成 **「场景一离线数据前置赋能 → 场景二实时通话毫秒级推理 → 分级策略智能干预 → 实时数据回流反哺场景一」** 的全链路双向闭环，实现离线与实时双场景数据的持续正向演化。
